Network Parsing, hands-on

This page turns one Android network request into something you can click through. Press Load weather and watch the app ask Open-Meteo for data, receive nested JSON, flatten it into one clean Kotlin object, and draw either a spinner, a weather card, or an error. The goal is simple: understand where the data goes, where errors are caught, and why the screen only needs one state value.

Retrofit kotlinx.serialization ViewModel + StateFlow Loading / Success / Error runnable + this explainer
1. Ask

The screen calls viewModel.loadWeather() and immediately shows Loading.

2. Fetch + parse

The Repository gets JSON from the fake data or the real Open-Meteo API. Serialization turns it into a nested ForecastResponseDto.

3. Draw

The ViewModel publishes Success(weather) or Error(message), and the screen redraws that state.

One sentence to hold onto The screen does not know about HTTP or JSON. It only renders one of three values: Loading, Success(weather), or Error(message).
If you get lost, trace these five nouns JSON is raw server text. ForecastResponseDto is the parsed, nested network shape. Weather is the flat app shape. WeatherUiState is what the screen renders. StateFlow is how the ViewModel publishes the current state to Compose.

Weather.kt — nested DTOs + flat domain + mapper
@Serializable                                  // makes this class JSON-decodable
data class ForecastResponseDto(
    val current: CurrentDto,                   // matches JSON "current": { … } (a nested object)
)

@Serializable
data class CurrentDto(
    @SerialName("temperature_2m") val temperature: Double,      // snake_case JSON → camelCase Kotlin
    @SerialName("apparent_temperature") val feelsLike: Double,
    @SerialName("relative_humidity_2m") val humidity: Int,
    @SerialName("weather_code") val weatherCode: Int,           // a NUMBER (WMO code), not text
    @SerialName("wind_speed_10m") val windSpeed: Double,
)

// PLAIN, FLAT data class: no @Serializable, no nesting, no cryptic weather_code.
data class Weather(
    val city: String, val country: String, val temperatureC: Int, val feelsLikeC: Int,
    val description: String, val humidityPercent: Int, val windKmh: Int, val weatherCode: Int,
)

// The SINGLE place the nested network-shape becomes the flat app-shape.
fun ForecastResponseDto.toWeather(city: City): Weather = Weather(
    city = city.name, country = city.country,
    temperatureC = current.temperature.roundToInt(),           // reach into "current", round
    feelsLikeC = current.feelsLike.roundToInt(),
    description = wmoDescription(current.weatherCode),          // 0 → "clear sky"
    humidityPercent = current.humidity, windKmh = current.windSpeed.roundToInt(),
    weatherCode = current.weatherCode,
)

WeatherApi.kt — the Retrofit interface + assembly
interface WeatherApi {
    @GET("v1/forecast")                        // relative path -> HTTP GET
    suspend fun fetchForecast(
        @Query("latitude") latitude: Double,   // becomes ?latitude=28.5384
        @Query("longitude") longitude: Double, // becomes &longitude=-81.3789
        @Query("current") current: String = "temperature_2m,…,wind_speed_10m",
    ): ForecastResponseDto                      // JSON object -> DTO, parsed for you
}

private val json = Json { ignoreUnknownKeys = true }   // drops the many fields we don't model

fun provideWeatherApi(): WeatherApi =
    Retrofit.Builder()
        .baseUrl("https://api.open-meteo.com/")                    // MUST end in "/" · no API key
        .addConverterFactory(json.asConverterFactory("application/json".toMediaType()))
        .build()
        .create(WeatherApi::class.java)        // Retrofit generates the implementation

WeatherRepository.kt — Real, Fake, switch
interface WeatherRepository { suspend fun getWeather(city: City): Weather }   // may THROW on failure

class RealWeatherRepository(private val api: WeatherApi = provideWeatherApi()) : WeatherRepository {
    override suspend fun getWeather(city: City): Weather {
        val dto = api.fetchForecast(city.latitude, city.longitude)   // network + parse
        return dto.toWeather(city)                                   // nested DTO → flat domain
    }
}

class FakeWeatherRepository : WeatherRepository {
    private val json = Json { ignoreUnknownKeys = true }
    override suspend fun getWeather(city: City): Weather {
        delay(600)                                              // imitate latency (see Loading)
        return json.decodeFromString<ForecastResponseDto>(FAKE_FORECAST_JSON).toWeather(city)
    }
}

// THE SWITCH — one argument toggles live (default, free + keyless) vs offline.
fun provideWeatherRepository(useFake: Boolean = false): WeatherRepository =
    if (useFake) FakeWeatherRepository() else RealWeatherRepository()

WeatherViewModel.kt — sealed state + loadWeather()
sealed interface WeatherUiState {
    data object Loading : WeatherUiState
    data class Success(val weather: Weather) : WeatherUiState
    data class Error(val message: String) : WeatherUiState
}

class WeatherViewModel(private val repository: WeatherRepository = provideWeatherRepository()) : ViewModel() {
    var city: City = PRESET_CITIES.first(); private set
    private val _uiState = MutableStateFlow<WeatherUiState>(WeatherUiState.Loading)
    val uiState: StateFlow<WeatherUiState> = _uiState.asStateFlow()
    init { loadWeather() }

    fun loadWeather(city: City = this.city) {
        this.city = city
        viewModelScope.launch {
            _uiState.value = WeatherUiState.Loading
            try { _uiState.value = WeatherUiState.Success(repository.getWeather(city)) }
            catch (e: Exception) { _uiState.value = WeatherUiState.Error(e.message ?: "Something went wrong while loading the weather.") }
        }
    }
}

MainActivity.kt — collect + exhaustive when
@Composable
fun WeatherScreen(modifier: Modifier = Modifier, viewModel: WeatherViewModel = viewModel()) {
    val uiState by viewModel.uiState.collectAsStateWithLifecycle()
    CityChips(selected = viewModel.city, onSelect = { viewModel.loadWeather(it) })
    WeatherContent(uiState, onRetry = { viewModel.loadWeather() }, modifier)
}

@Composable
fun WeatherContent(uiState: WeatherUiState, onRetry: () -> Unit, modifier: Modifier = Modifier) {
    when (uiState) {                                           // exhaustive: sealed type
        is WeatherUiState.Loading -> LoadingState(modifier)
        is WeatherUiState.Success -> WeatherCard(uiState.weather, modifier)
        is WeatherUiState.Error   -> ErrorState(uiState.message, onRetry, modifier)
    }
}

Key idea · the parsing boundary

DTO → domain mapping

JSON arrives in the network's nested shape (ForecastResponseDto wrapping a CurrentDto); the app uses its own clean, flat shape (Weather). The mapper toWeather() is the single boundary — it reaches into current, rounds the numbers, and decodes weather_code into text. Prompt 1: fake vs real

If Fake and Real both implement WeatherRepository, what code does the ViewModel need to change?

Reveal

None. It calls repository.getWeather(city) either way. The factory chooses the implementation.

Prompt 2: extra field

If the response adds "current_units": {…}, should the UI show Error?

Reveal

No. ignoreUnknownKeys = true lets parsing succeed, and toWeather() still outputs the same clean Weather.

Prompt 3: wrong type

If temperature_2m arrives as a String instead of a number, which layer turns that into UI?

Reveal

The parser throws, then the ViewModel's catch publishes Error(message).

Prompt 4: no manual refresh

Why does the weather card appear even though no code calls a UI refresh method?

Reveal

The ViewModel changes uiState. Compose observes it with collectAsStateWithLifecycle() and recomposes the Success branch.

Common mistake: DTOs in UI

Symptom: screen code starts depending on wire-only fields like the nested current.weather_code number.

Fix

Keep DTOs at the network boundary and flatten them into domain Weather objects (with weather_code already decoded) before UI state is published.

Common mistake: previewing a ViewModel

Symptom: previews become flaky because they try to run loading logic.

Fix

Preview the stateless WeatherContent() with hand-made Loading, Success, and Error states.

Teacher note Have students run the simulator after each reveal. The goal is to connect the answer to a visible state change, not just read the explanation.
