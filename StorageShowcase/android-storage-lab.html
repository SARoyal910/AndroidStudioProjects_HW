The useful mental model

Android storage is a set of scoped places, not one shared disk.

Every storage choice answers five questions: who owns the data, who should see it, how structured it is, whether it should survive uninstall, and whether the user should explicitly choose the file or media.

Rule of thumb: default to the narrowest storage surface that satisfies the feature. Private app state belongs in the app sandbox. User-visible media belongs in MediaStore. Arbitrary user documents belong behind the system picker.
Private app data Room, DataStore, filesDir, cacheDir. Other apps cannot browse it. Removed when the app is uninstalled.
Shared user data Photos, videos, audio, downloads, and documents. Use system APIs so Android mediates access.
Granted access Photo Picker and SAF return content URIs. Your app receives access to exactly what the user selected.

Permissions by API level

Modern storage access is mostly URI and collection based.

Use the sliders to compare device API level, target SDK, and operation. Android 13 introduced granular media permissions. Android 14 added selected photo/video access in the permission prompt. Photo Picker and SAF avoid storage permissions for user-selected content.

Room and SQLite

Room stores structured app data and emits reactive query results.

Use Room when data has rows, relationships, queries, sorting, transactions, or migrations. A DAO can return Flow<List<Note>>; writes invalidate the query and the UI receives a new list.

DataStore

            
DataStore is for small private state, read as a Flow and written transactionally.

            
Preferences DataStore stores key-value preferences. Proto or typed DataStore stores one typed object. Both are asynchronous and transactional, which makes them a better default than legacy SharedPreferences for new code.

Files and cache

            
Internal files are private bytes; cache is disposable bytes.

            
Use files when the app owns the bytes and does not need table queries. Use cache only for data you can rebuild. Raw file I/O is blocking, so Android apps should move it off the main thread.

MediaStore

            
Shared photos, videos, and audio are addressed through collections and URIs.

            
MediaStore lets your app add user-visible media and query media collections. On Android 10 and higher, your app can add its own media without storage permission. Reading other apps' media depends on Photo Picker or media permissions.

Storage Access Framework

            
SAF is the user-controlled path for arbitrary documents.

            
SAF returns a content URI from a system picker. Your app does not need storage permission because the user selects the document or directory. Android 11 and higher restrict broad tree access to sensitive roots such as storage root, Downloads, Android/data, and Android/obb.

Backup, privacy, and security

            
Persistence is not the same thing as secrecy.

            
Android's app sandbox prevents casual cross-app reads, but your own storage files are still plain app data unless you encrypt them. Backup behavior is a separate policy decision.
