SupabaseSync, hands-on

SupabaseSync shows cloud storage used together with Room — the offline-first pattern. The real cloud here is Supabase — a hosted Postgres database reached via the supabase-kt client; Room is the offline single source of truth (Supabase has no built-in local cache), and a background sync reconciles Room's notes table with the Postgres notes table. Everything keeps working offline. This page lets you drive that yourself: toggle the network, add and delete notes, and watch them queue as Pending and drain to Synced — then explore conflicts, the architecture, and the source.

What Flow does Flow is how Room keeps the screen updated. A DAO function like observeVisibleNotes(): Flow<List<Note>> does not mean "give me the list once." It means "send me the current list, then send me a new list any time the matching database rows change." When the app adds a note, deletes a note, or sync pulls a cloud note into Room, the Flow emits again and the UI redraws from Room. A StateFlow is the screen-ready version of that stream: it always has the latest value available, so Compose can draw immediately.

data/Note.kt — entity + DTO + sync fields
enum class SyncState { SYNCED, PENDING, FAILED }    // the outbox marker

@Entity(tableName = "notes")
data class Note(
    @PrimaryKey val id: String,            // CLIENT-generated UUID (an offline row has an id already)
    val title: String, val body: String,
    val updatedAt: Long,                   // last-write-wins clock
    val syncState: SyncState = SyncState.PENDING,
    val deleted: Boolean = false,          // soft-delete tombstone (pushable while offline)
)

@Serializable
data class NoteDto(val id: String, val title: String, val body: String, val updatedAt: Long, val deleted: Boolean = false)

fun Note.toDto() = NoteDto(id, title, body, updatedAt, deleted)
fun NoteDto.toEntity(state: SyncState = SyncState.SYNCED) = Note(id, title, body, updatedAt, state, deleted)

@Dao
interface NoteDao {
    @Query("SELECT * FROM notes WHERE deleted = 0 ORDER BY updatedAt DESC")
    fun observeVisibleNotes(): Flow<List<Note>>            // tombstones hidden

    @Query("SELECT COUNT(*) FROM notes WHERE syncState != 'SYNCED'")
    fun observePendingCount(): Flow<Int>

    @Query("SELECT MAX(updatedAt) FROM notes")
    suspend fun maxUpdatedAt(): Long?                       // pull cursor

    @Query("SELECT * FROM notes WHERE syncState != 'SYNCED'")
    suspend fun notesToPush(): List<Note>                   // the OUTBOX

    @Insert(onConflict = OnConflictStrategy.REPLACE) suspend fun upsert(note: Note)
    @Query("DELETE FROM notes WHERE id = :id") suspend fun hardDelete(id: String)

data/CloudApi.kt — Supabase (Real) vs Fake + the switch
interface CloudApi {
    suspend fun pullSince(since: Long): List<NoteDto>
    suspend fun push(note: NoteDto): NoteDto
}

// REAL — Supabase (hosted Postgres) via supabase-kt + Postgrest
class SupabaseCloudApi(url: String, key: String) : CloudApi {
    private val client = createSupabaseClient(url, key) { install(Postgrest) }
    override suspend fun pullSince(since: Long): List<NoteDto> =
        client.from("notes").select {                         // SELECT * FROM notes WHERE ...
            filter { gt("updated_at", since) }                // updated_at > cursor
            order("updated_at", Order.ASCENDING)
        }.decodeList<NoteDto>()
    override suspend fun push(note: NoteDto): NoteDto =
        client.from("notes").upsert(note) { select() }        // INSERT ... ON CONFLICT DO UPDATE
            .decodeSingle<NoteDto>()                          // RETURNING the stored row
}

fun provideCloudApi(useFake: Boolean = true): CloudApi =      // THE SWITCH
    if (useFake) fakeCloud else SupabaseCloudApi(SUPABASE_URL, SUPABASE_ANON_KEY)

data/NoteRepository.kt — optimistic writes + sync() + last-write-wins
// PURE last-write-wins rule (unit-tested with no Room/coroutines).
fun shouldAcceptRemote(local: Note?, remote: NoteDto): Boolean =
    local == null || (local.syncState == SyncState.SYNCED && remote.updatedAt >= local.updatedAt)

suspend fun addNote(title: String, body: String) {
    dao.upsert(Note(UUID.randomUUID().toString(), title.trim(), body.trim(),
        System.currentTimeMillis(), SyncState.PENDING))    // optimistic, joins the outbox
}

suspend fun sync() {
    for (local in dao.notesToPush()) {                     // 1) PUSH the outbox
        val serverCopy = cloud.push(local.toDto())         // on failure: mark FAILED + rethrow → worker retries
        if (local.deleted) dao.hardDelete(local.id) else dao.upsert(serverCopy.toEntity(SyncState.SYNCED))
    }
    val since = dao.maxUpdatedAt() ?: 0L                   // 2) PULL remote changes
    for (remote in cloud.pullSince(since)) {
        if (shouldAcceptRemote(dao.findById(remote.id), remote))
            if (remote.deleted) dao.hardDelete(remote.id) else dao.upsert(remote.toEntity(SyncState.SYNCED))
    }
}

sync/SyncWorker.kt + SyncScheduler.kt — background, network-gated
class SyncWorker(appContext: Context, params: WorkerParameters) : CoroutineWorker(appContext, params) {
    override suspend fun doWork(): Result {
        val repo = NoteRepository.get(applicationContext)   // the SAME repo the UI uses
        return try { repo.sync(); Result.success() }
        catch (e: Exception) { Result.retry() }            // backoff; try again later
    }
}

// The constraint is WHY offline just queues:
OneTimeWorkRequestBuilder<SyncWorker>()
    .setConstraints(Constraints.Builder()
        .setRequiredNetworkType(NetworkType.CONNECTED)     // run online, DEFER offline
        .build())
    .setBackoffCriteria(BackoffPolicy.EXPONENTIAL, 30, TimeUnit.SECONDS)
    .build()

Key idea · who wins?

Conflict resolution — last-write-wins

Both devices edit the same note while offline, then both sync. Something must decide whose version survives. SupabaseSync compares updatedAt — the later edit wins. Make each device edit and watch the rule (shouldAcceptRemote) pick a winner.

Key idea · one-line offline switch

Real vs Fake — the same interface, two clouds

Both implement CloudApi, so the repository can't tell them apart. provideCloudApi(useFake = …) picks one in a single place — the real impl is SupabaseCloudApi (a hosted Postgres database via supabase-kt), and the fake is an offline in-memory server.
