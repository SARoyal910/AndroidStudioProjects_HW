One idea carries the whole game: state + pure rules + a UI that draws the state The board is an immutable value (Board). Every rule — build, flip, reset — is a pure function that returns a new board rather than editing the old one. The screen holds the current board in remember { mutableStateOf(...) }; a tap calls a rule function and stores the returned board back into that state, which is exactly what makes Compose redraw. Because the rules are pure and Android-free, the unit tests drive them on the JVM with no emulator. Get that loop right and the game almost writes itself.

Part B. The game logic — MatchLogic.kt

Pure Kotlin, no Android imports. The model (TileState/Tile/Board and its helpers) is provided; you implement the three functions below.

B2 — Build a fresh, shuffled boardAndroid Studio

Return a board of PAIR_COUNT (8) symbols, each used twice, shuffled into the 16 positions, all FaceDown, with moves = 0. Accept a Random so tests can seed it.

Where: MatchLogic.kt → newBoard()

fun newBoard(symbols: List<String>, random: Random = Random.Default): Board {
    // 1. Take the first PAIR_COUNT (8) symbols and DOUBLE the list (+ joins two
    //    lists), so every symbol now appears exactly twice -> 16 faces in all.
    // 2. shuffled(random) returns a NEW randomly-ordered copy. We pass `random`
    //    in (rather than calling Random.Default here) so a test can hand us a
    //    seeded Random and get a repeatable board — that's the testability hook.
    // 3. map wraps each face in a FaceDown Tile.
    val tiles = (symbols.take(PAIR_COUNT) + symbols.take(PAIR_COUNT))
        .shuffled(random)
        .map { face -> Tile(face, TileState.FaceDown) }
    return Board(tiles = tiles, moves = 0)
}

Why it works: doubling the 8-symbol list yields the 16 faces a 4×4 board needs with each symbol present exactly twice; shuffled(random) randomizes their positions while threading the caller's RNG through, so production gets real randomness and tests get a repeatable layout from Random(seed). The Board(tiles, moves = 0) constructor's init check passes because the list is exactly 16 long.

Common mistake: calling symbols.shuffled().take(16) instead of doubling — that picks 16 distinct symbols with no pairs, so nothing can ever match. Or ignoring the random parameter and calling shuffled() with no argument, which makes the determinism test fail.
B3 — Flip a tile (the heart of the game)Android Studio

Apply the tap rules and return a new board. The outcome depends on how many tiles are currently face-up (0, 1, or 2).

Where: MatchLogic.kt → Board.flip() (plus a small withState helper)

fun Board.flip(index: Int): Board {
    val up = faceUpIndices() // the 0, 1, or 2 tiles currently showing their face.

    return when {
        // (a) A leftover, non-matching pair is still showing from the previous
        //     turn. This tap's ONLY job is to clear it: flip those two back down.
        //     We do NOT reveal the tapped tile and we do NOT count a move.
        up.size == 2 ->
            withState(up.toSet(), TileState.FaceDown)

        // (b) No-op: an out-of-range index, or a tile already FaceUp/Matched.
        //     Only FaceDown tiles can be revealed. Hand back the unchanged board.
        index !in tiles.indices || tiles[index].state != TileState.FaceDown ->
            this

        // (c) Nothing is up yet -> simply reveal the tapped tile.
        up.isEmpty() ->
            withState(setOf(index), TileState.FaceUp)

        // (d) Exactly one tile is up -> reveal the tapped one to make two, then
        //     judge the pair. Either way a pair ATTEMPT happened, so moves + 1.
        else -> {
            val other = up.first()
            val isMatch = tiles[index].face == tiles[other].face
            val newState = if (isMatch) TileState.Matched else TileState.FaceUp
            withState(setOf(index, other), newState, moveDelta = 1)
        }
    }
}

// Return a NEW board with the tiles at [indices] set to [state] and moves bumped
// by [moveDelta]. mapIndexed builds a fresh List; untouched tiles are reused as-is,
// so the original board is never mutated.
private fun Board.withState(indices: Set<Int>, state: TileState, moveDelta: Int = 0): Board =
    copy(
        tiles = tiles.mapIndexed { i, tile ->
            if (i in indices) tile.copy(state = state) else tile
        },
        moves = moves + moveDelta,
    )

Why it works: branching on faceUpIndices().size turns the English rules into four mutually-exclusive cases, ordered most-specific first. Case (a) clears a leftover mismatch before anything else so a stale pair can't block the next turn; (b) makes illegal taps harmless; (c) reveals the first tile of a pair; (d) reveals the second, compares faces, and locks (Matched) or shows (FaceUp) accordingly, counting the one move. Every branch funnels through withState, which rebuilds the list with mapIndexed+copy — so a brand-new Board comes out and the one passed in is untouched.

Common mistake: mutating a tile in place (e.g. a MutableList and tiles[i] = ...) — Compose won't see a change and the immutability test fails. Also: counting a move on the first reveal (case c), or incrementing moves when clearing a mismatch (case a). A move is one pair attempt, i.e. only case (d).
B4 — Reset (clear) the boardAndroid Studio

Keep the same faces in the same positions, but turn every tile FaceDown and zero the move counter.

Where: MatchLogic.kt → Board.reset()

fun Board.reset(): Board =
    copy(tiles = tiles.map { it.copy(state = TileState.FaceDown) }, moves = 0)

Why it works: copy(...) on the data class produces a new Board; tiles.map { it.copy(state = FaceDown) } produces a new list where every tile keeps its face (and its index, since map preserves order) but loses its revealed/matched state. That is exactly "same layout, fresh start" — distinct from New Game, which calls newBoard(...) for a brand-new shuffle.

Common mistake: reshuffling inside reset() — that's what New Game is for. Reset must preserve the order so the player can replay the same board.

C2 — The 4×4 board of tappable tilesAndroid Studio

Render the 16 tiles as a 4-column grid of square, tappable cells whose look depends on each tile's state.

Where: MainActivity.kt → BoardGrid + TileView

@Composable
private fun BoardGrid(board: Board, onTap: (Int) -> Unit, modifier: Modifier = Modifier) {
    LazyVerticalGrid(
        columns = GridCells.Fixed(GRID_SIZE),                 // 4 columns -> a 4×4 layout
        modifier = modifier,
        horizontalArrangement = Arrangement.spacedBy(8.dp),
        verticalArrangement = Arrangement.spacedBy(8.dp),
        userScrollEnabled = false,                            // the whole board fits on screen
    ) {
        itemsIndexed(board.tiles) { index, tile ->
            TileView(tile = tile, onTap = { onTap(index) })
        }
    }
}

@Composable
private fun TileView(tile: Tile, onTap: () -> Unit) {
    val background = when (tile.state) {
        TileState.Matched -> MatchedColor                            // green: a found pair
        TileState.FaceUp -> FaceUpColor                              // blue: just revealed
        TileState.FaceDown -> MaterialTheme.colorScheme.surfaceVariant
    }
    val faceText = if (tile.state == TileState.FaceDown) "?" else tile.face
    val faceColor = if (tile.state == TileState.FaceDown)
        MaterialTheme.colorScheme.onSurfaceVariant else Color.White

    Card(
        onClick = onTap,                              // whole face is tappable + accessible
        modifier = Modifier.aspectRatio(1f),          // force a perfect square
        shape = RoundedCornerShape(10.dp),
        colors = CardDefaults.cardColors(containerColor = background),
        border = BorderStroke(1.dp, MaterialTheme.colorScheme.outlineVariant),
    ) {
        Box(modifier = Modifier.fillMaxSize(), contentAlignment = Alignment.Center) {
            Text(text = faceText, fontSize = 30.sp, color = faceColor)
        }
    }
}

Why it works: LazyVerticalGrid(GridCells.Fixed(GRID_SIZE)) arranges the flat 16-tile list into 4 columns automatically — every 5th item wraps to a new row. itemsIndexed hands back both the tile and its 0..15 index, which the tap forwards up so the screen knows which tile was pressed. aspectRatio(1f) keeps cells square at any width, and the when over tile.state gives each of the three states a distinct, glanceable look.

Common mistake: using items(board.tiles) (no index) and then having no way to tell the screen which tile was tapped. Also, leaving the grid scrollable or not setting aspectRatio(1f), which makes the cells stretch into rectangles.
C3 — The win bannerAndroid Studio

When the board is solved, show a celebratory message that includes the move count.

Where: MainActivity.kt → WinBanner (shown when won)

if (won) {

@Composable
private fun WinBanner(moves: Int) {
    Text(
        text = "🎉  You win in $moves moves!",
        modifier = Modifier
            .fillMaxWidth()
            .clip(RoundedCornerShape(12.dp))          // round corners BEFORE the background…
            .background(MatchedColor)                 // …so the fill respects the shape
            .padding(vertical = 12.dp, horizontal = 16.dp),
        color = Color.White,
        fontWeight = FontWeight.Bold,
        textAlign = TextAlign.Center,
    )
}

Why it works: won is derived from board.isSolved(), so the banner appears automatically the instant the last pair is matched and disappears after New Game / Reset — no extra state to manage. Ordering clip before background makes the colored fill follow the rounded shape.

Common mistake: tracking a separate var hasWon boolean and forgetting to reset it on New Game, so the banner sticks around. Deriving it from the board avoids the bug entirely.

Part D. Unit tests — MatchLogicTest.kt

Pure rules mean fast JVM tests. The full key has 11 tests (all passing). Below are the test helpers and the trickiest cases — match, mismatch-then-clear, and a full win — that pin down the rules.

D — The helpers and the key testsAndroid Studio

Verify generation, flipping (reveal / match / mismatch-then-clear), immutability, win, and reset.

Where: MatchLogicTest.kt — run with ./gradlew testDebugUnitTest

// Find the OTHER tile sharing a face with tile [i] (its twin) — there's exactly one.
private fun Board.twinOf(i: Int): Int =
    tiles.indices.first { it != i && tiles[it].face == tiles[i].face }

// Find any tile whose face differs from tile [i]'s — used to force a mismatch.
private fun Board.differentFrom(i: Int): Int =
    tiles.indices.first { tiles[it].face != tiles[i].face }

// Play the whole board to completion, matching one pair at a time.
private fun solved(seed: Int): Board {
    var b = newBoard(DEFAULT_SYMBOLS, Random(seed))
    repeat(PAIR_COUNT) {
        val i = b.tiles.indexOfFirst { it.state == TileState.FaceDown }
        b = b.flip(i).flip(b.twinOf(i)) // reveal one, then its twin -> both Matched
    }
    return b
}

@Test
fun flip_matchingPair_locksBoth_andCountsOneMove() {
    val fresh = newBoard(DEFAULT_SYMBOLS, Random(3))
    val i = 0
    val j = fresh.twinOf(i)
    val b = fresh.flip(i).flip(j)
    assertEquals(TileState.Matched, b.tiles[i].state)
    assertEquals(TileState.Matched, b.tiles[j].state)
    assertEquals(1, b.moves)
}

@Test
fun flip_mismatch_leavesBothUp_thenNextTapClearsThem() {
    val fresh = newBoard(DEFAULT_SYMBOLS, Random(5))
    val i = 0
    val j = fresh.differentFrom(i)
    val two = fresh.flip(i).flip(j)
    assertEquals(TileState.FaceUp, two.tiles[i].state)   // both stay up so the miss is visible
    assertEquals(TileState.FaceUp, two.tiles[j].state)
    assertEquals(1, two.moves)

    val k = two.tiles.indices.first { it != i && it != j }
    val cleared = two.flip(k)
    assertEquals(TileState.FaceDown, cleared.tiles[i].state)
    assertEquals(TileState.FaceDown, cleared.tiles[j].state)
    assertEquals(TileState.FaceDown, cleared.tiles[k].state) // k was NOT revealed
    assertEquals(1, cleared.moves)                           // clearing is not a move
}

@Test
fun flip_isImmutable_doesNotMutateTheOriginalBoard() {
    val original = newBoard(DEFAULT_SYMBOLS)
    original.flip(0)                                  // discard the result
    assertEquals(TileState.FaceDown, original.tiles[0].state)
}

@Test
fun matchingEveryPair_winsWithExactlyPairCountMoves() {
    val b = solved(7)
    assertTrue(b.isSolved())
    assertEquals(PAIR_COUNT, b.moves)                 // 8 pairs -> 8 moves on a perfect game
}

Why it works: seeding Random makes each board reproducible, so assertions are stable. twinOf and differentFrom let a test deterministically force a match or a mismatch without knowing the shuffle. The mismatch test is the important one — it checks both halves of the "clear on the next tap" rule (two up after the miss, both down after the next tap, and that the next tap reveals nothing and adds no move). solved() drives a whole game so the win and reset tests have a fully-played board to assert against.

Common mistake: hard-coding tile indices like "0 and 1 match" — the board is shuffled, so 0 and 1 usually don't match. Always find the twin by face. The other 7 tests in the suite cover size, the 8-distinct-faces-each-twice composition, determinism, the no-move first reveal, out-of-range/already-up no-ops, and reset.
